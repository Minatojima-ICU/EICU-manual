# ターミナルの練習

このページでは、最低限のターミナル操作を実際に手を動かして練習します。

## 0. ターミナルとは
ターミナルは、デバイスと対話するためのツールです。
ターミナルからデバイスへ命令して、デバイスからの反応を確認します。  


### CLIとGUI
普段使っているwindowsやmacでは、マウス/トラックパッドを使って操作をします。この視覚的操作を行うための仕組みをGUI (Graphical User Interface)と言います。  
対して、ターミナル上でコマンド（命令）を文字で入力してパソコンを操作する仕組みをCLI (Command Line Interface)と言います。  
直感ではGUIの方が絶対に使いやすいと思うのですが、CLIに慣れてくるとその使い所がわかってきます。

## 1. 練習用フォルダを作る
* ターミナルを使ったことがない方は、任意の場所に `terminal-practice` というフォルダ（ディレクトリといいます）を作ってください

* すでにターミナルを使ったことがある方は、任意の場所で以下を実行してください。
```bash
mkdir terminal-practice
```

## 2. 履修コマンド
ターミナルを実行して、出てきたウィンドウの一番最後にカーソルがあることを確認してください。

### `ls` （一覧を見る） (必修) 

現在のディレクトリにあるファイル/ディレクトリの一覧を表示します。("list" の略)

#### 演習 (基礎)
`ls` を実行して、現在のディレクトリの中身を表示しましょう。

```bash
ls
```
#### 演習 (発展)
コマンド（命令のことです）は、その後にオプションをつけることで機能を変更することができます。  
`ls -a` を実行して、現在のディレクトリの中身をすべて表示しましょう。これにより隠しファイル/ディレクトリも表示することができます。


```bash
ls -a
```
なお、以下も同等の動きをします。

```bash
ls --all
```

"-"（ハイフン）が1つの場合は短い単語で記述でき、2つの場合は長い単語で記述します。

---

### `cd` （移動する） (必修)

ディレクトリの移動に使います。フォルダをダブルクリックしてその中に入っていくのと同じ動作です。("change directory"の略)  

#### 演習（基礎）
まず `ls` で現在のディレクトリ内のファイル/ディレクトリを表示します。そのうち、いずれかのディレクトリ名に対して `cd` を実行してみましょう。 

```bash
ls
cd Documents #これはあなたが指定してください。
```
これで、指定したディレクトリの中に移動できました。 `ls` を実行して中身を確認しましょう。

```bash
ls
```
中身が確認できたと思います。

次に、一つ上のディレクトリに移動しましょう。要するに"戻る"ということです。

`cd ..` で1つ上に戻ります。スペース一つのあとに、ピリオドを2つつけます。

```bash
cd ..
```
ピリオド2つは、上に行くということです。

一気にホーム（一番最初の位置）に移動するには、 `cd` だけを実行します。  
どこにいてもこれで一気にホームに戻ります。

```bash
cd
```

#### 演習（発展）
移動先のディレクトリ名のタイピングを省略することができます。  
`cd` と入力し、スペースをうってTABキーを押すと、移動先のリストが表示されます。そしてさらにTABキーを押していくと移動先を指定することができます。  
また、途中までファイル名/ディレクトリ名を入力してそこから入力内容を省略することも可能です。

```bash
cd Do # ここでTABを押すと、移動先の選択肢を少なくすることができる（"Documents"と"Downloads"、など）
```

その他、`cd` でよく使う概念は、相対パス、絶対パスになります。必要に応じて調べてみてください。

---

### `mkdir` （ディレクトリを作る） (必修)

`mkdir` で新しくディレクトリを作ることができます。（"make directory"の略）

#### 演習（基礎）
それでは、一番最初に作った `terminal-practice` ディレクトリに移動してください。  
その中で、`mkdir` を使って新しいディレクトリをターミナル上で作りましょう。  
`notes` というディレクトリを作って、 `ls` でディレクトリの存在を確認します。

```bash
mkdir notes # 空のディレクトリを作る
ls          # 実際にディレクトリが作られているか確認する
```

---

### `touch` （空ファイルを作る） (必修)

`touch`を使って空ファイルを作ります。

#### 演習（基礎）
先ほど作成した `notes` というディレクトリに移動した後、`memo.txt` という空のファイルを作成して、 `ls` でファイルの存在を確認します。

```bash
cd notes
touch memo.txt
ls
```

### Nanoの使い方 （発展） 

`nano` はターミナルで使えるシンプルなテキストエディタです。ターミナル上でテキストを編集できます。  
`nano` 以外には、vi、vim、nvim、emacsなどがあります。初学者には `nano` がとっつきやすいと言われています(私はnvimを使っています)。  
ターミナル上でテキストエディタを展開する利点は、わざわざ他のアプリを立ち上げずにターミナル上で操作がすべてできることです。欠点は、操作に慣れる必要があるということです。  
`nano` を使わなくても普通のテキストエディタを使えばいいので、`nano` の操作についての学習は飛ばしてもらっても構いません。

* 以下の演習を行わない場合は、`memo.txt` をいつも使っているテキストエディタで開いて、一行目に
```
Hello world
```
と記載してください。

#### 演習（発展）
##### テキストファイルを開く

nanoを使ってmemo.txtを開きましょう。

```bash
nano memo.txt
```

コマンドを実行すると、ターミナル上で `nano` が起動します。  
もし `memo.txt` が存在しない場合は、新しいファイルとして開かれます。

画面の下部には、`^` で表されるショートカットキーの一覧が表示されています。  
ここでの `^` は **Ctrlキー** を意味します。  

例えば、

```
^X Exit
```

と書かれている場合は **Ctrl + X** を押すと終了できる、という意味です。


##### テキストを書いてみる

`nano` が起動したら、そのままキーボードで文字を入力できます。  
次の文字列を入力してみましょう。

```
Hello world
```

入力は通常のテキストエディタと同じように行えます。  
矢印キーでカーソルを移動することもできます。


##### ファイルを保存する

入力できたら、ファイルを保存します。

1. **Ctrl + O** を押します（Write Out = 保存）

画面下に次のような表示が出ます。

```
File Name to Write: memo.txt
```

そのまま **Enter** を押します。  
これで `memo.txt` が保存されます。


##### nano を終了する

保存できたら、`nano` を終了します。

1. **Ctrl + X** を押します（Exit）

これで `nano` が終了し、元のターミナル画面に戻ります。



##### まとめ

今回覚える `nano` の基本操作は次の3つです。

| 操作 | キー |
|---|---|
| 保存 | Ctrl + O |
| 決定 | Enter |
| 終了 | Ctrl + X |

まずはこの3つだけ覚えれば、`nano` を使ってファイルを作成・編集できます。

#### おまけ
ターミナルから、いつも使っているテキストエディタを立ち上げることができます。`nano` を使わない時は、以下で対応してもよいかと思います。

```bash
# Macであればこちら
open -a TextEdit memo.txt

# Windowsはこっち
notepad memo.txt

# Visual Studio Codeで編集する場合
code memo.txt
```

### `cat` （ファイル内容を表示する） (基礎)
`cat` を使うと、ターミナル上でテキストファイルの中身を見ることができます。  

#### 演習 （基礎）

```bash
cat memo.txt
```
"Hello world"が出力されることを確認してください。

---

### `rm` （削除する） (発展) 
`rm` でファイルを削除することができます。  

**重要!!** <u>`rm` で削除したファイルは復元できません!</u>　ゴミ箱を経由せず削除されるからです。一瞬で削除されてしまいます。慣れるまでは、`rm` で削除しない方がいいかもしれません。

#### 演習（発展）
```bash
rm memo.txt
ls
```
`memo.txt` が一覧から消えていれば成功です。

`rm` はディレクトリも削除できます。ポイントは `-r`（再帰的に削除）です。  

まず練習用ディレクトリを作ります。

```bash
mkdir -p tmp_rm/subdir
touch tmp_rm/file1.txt tmp_rm/subdir/file2.txt
ls -R tmp_rm
```

次に、`rm` だけでディレクトリ削除を試してエラーになることを確認します。

```bash
rm tmp_rm
```

`-r` をつけるとディレクトリごと削除できます。

```bash
rm -r tmp_rm
ls
```

`tmp_rm` が消えていれば成功です。
