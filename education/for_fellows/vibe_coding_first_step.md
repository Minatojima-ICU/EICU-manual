# AIコーディングエージェント入門（初心者向け）

このガイドは、バイブコーディング（自然言語でAIに開発を手伝ってもらうこと）を初めて試す人向けです。  
以前はターミナルを使う準備が必要でしたが、現在はCodex、Claude、AntigravityなどのデスクトップアプリやWeb版から始められます。

最初の目的は「無料枠で、安全に1回だけ試してみる」ことです。  
本格的な開発や課金設定は、最初の練習が終わってから考えます。

## 1. このページでできるようになること

- AIコーディングエージェントの基本的な使い方を知る
- 練習用フォルダまたはプロジェクトを作る
- デスクトップアプリまたはWeb版で作業を始める
- AIに簡単なファイル作成を依頼する
- 作られたファイルを確認し、必要なら修正を依頼する
- 安全に使うための最低限のルールを知る

## 2. AIコーディングエージェントとは

AIコーディングエージェントは、チャットで指示すると、ファイルを読んだり、コードを書いたり、アプリを作ったりしてくれるツールです。

通常のChatGPTやClaudeとの違いは、単に回答するだけではなく、ファイル作成やコード作成まで手伝える点です。  
Codex appやAntigravityでは練習用フォルダを使い、Claudeではプロジェクト管理を使って安全に試します。

## 3. 最初に使うツール

初心者は、まず1つだけ選んでください。  
最初の1回目は、無料枠または既存サブスクリプションの範囲で試します。APIキーや従量課金の設定は不要です。

| ツール | 向いている人 | 特徴 |
|---|---|---|
| Codex app | ChatGPTを使っている人 | デスクトップアプリで始めやすい |
| Claude | Claudeを使っている人 | デスクトップアプリのプロジェクト管理で安全に始めやすい |
| Antigravity | Google/Gemini系のツールを使いたい人 | AIエージェント中心の開発環境。無料枠で試しやすい |

このページでは、ターミナルを使わない方法を基本にします。  
Codex CLI、Claude Code CLI、Gemini CLIなどのCLI版は、おまけとして後半に回します。

## 4. 事前準備

### 必要なもの

- インターネット接続
- PC（Windows または Mac）
- メールアドレス
- ChatGPT、Claude、Googleアカウントのいずれか
- 30〜60分の作業時間

### 注意

- 最初の1回目は無料枠で行ってください。
- APIキーを作る必要はありません。
- API利用は従量課金になることがあるため、初心者の最初の練習では使いません。
- 病院支給PCの場合は、院内ルールに従ってください。
- 患者情報、個人情報、院内資料、未公開資料は入力しないでください。

## 5. 練習用フォルダまたはプロジェクトを用意する

Codex appまたはAntigravityを使う場合は、AIに触らせてもよい練習用フォルダ `vibe_coding_test` を1つ作ります。
最初は、このフォルダ以外をAIに作業させないでください。

Claudeを使う場合は、PC上のフォルダではなく、Claudeデスクトップアプリ内に `vibe_coding_test` というプロジェクトを作って進めます。

*既存の大切なファイルが入っているフォルダを選ばないことが重要です。*

## 7. 最初の練習

以下は共通の練習内容です。  
実際に進めるときは、選んだツールごとのページにある手順を使ってください。

まずはコードではなく、短いテキストファイルを作ってもらいます。

```prompt
このフォルダに `ARDS診断基準.txt` というファイルを作ってください。

内容は、ARDSの診断基準を医療者向けの学習用に短くまとめたものにしてください。
患者情報や個人情報は含めないでください。
内容は初心者にも読みやすいようにしてください。
```

作成後、次を確認してください。

- `ARDS診断基準.txt` が作られている
- 内容が短くまとまっている
- 患者情報や個人情報が含まれていない
- 余計なファイルが作られていない

内容が長すぎる場合は、次のように追加で依頼します。

```prompt
少し長いので、初学者向けに半分くらいの長さにしてください。
箇条書きを中心にして、読みやすくしてください。
```

## 8. 次の練習: 簡単なWebアプリを作る

1回目のファイル作成ができたら、次に簡単なWebアプリを作ってみます。

```prompt
## 課題
ARDS患者の目標一回換気量を計算して表示するWebアプリを作ってください。

## 条件
- HTML, CSS, JavaScriptだけで作ってください
- 外部APIや生成AI APIは使わないでください
- 身長(cm)と性別を入力できるようにしてください
- Devineの式でIBWを計算してください
- 6-8 ml/kg IBWの一回換気量の範囲を表示してください
- 医療現場で使う正式ツールではなく、学習用であることを画面に表示してください
- 作成後、ブラウザで開く方法も教えてください

## Devineの式
- Male: IBW (kg) = 50 + 2.3 × ((height_cm / 2.54) - 60)
- Female: IBW (kg) = 45.5 + 2.3 × ((height_cm / 2.54) - 60)
```

作成後、以下を確認してください。

- 入力欄がある
- 計算結果が表示される
- 学習用であることが明記されている
- 余計な外部サービスが使われていない

## 9. 依頼するときのコツ

AIへの依頼には、次の情報を入れると成功しやすくなります。

- 目的
- 対象者
- 作ってほしいもの
- 使ってよい技術
- 使ってはいけないもの
- 出力形式
- 安全上の注意

悪い例:

```prompt
ARDSのアプリ作って
```

良い例:

```prompt
初心者向けの学習用として、ARDS患者の目標一回換気量を計算する簡単なWebアプリを作ってください。
HTML, CSS, JavaScriptだけを使い、外部サービスやAPIは使わないでください。
```

## 10. 安全ルール（重要）

- 患者情報を入力しない
- 個人情報、ID、パスワード、APIキーを貼らない
- 最初は練習用フォルダだけを開く
- 意味がわからない許可は出さない
- ファイル削除や大量変更の提案は、内容を確認してから許可する
- 生成AI APIをアプリ内に組み込まない
- 医療内容は必ず人間が確認する

特に、AIが「コマンドを実行してよいか」「ファイルを削除してよいか」と聞いてきた場合は、内容が理解できるまで許可しないでください。

## 11. うまくいかないとき

エラーが出たら、エラーメッセージをそのままAIに見せて相談してください。

よくある例:

### ログインできない

- アカウントが違っていないか確認する
- 無料枠または利用可能なプランか確認する
- ブラウザで同じサービスにログインできるか確認する

### フォルダを開けない

- デスクトップやドキュメントなど、自分で作ったフォルダを選ぶ
- 病院支給PCの場合、保存先の制限がないか確認する

### ファイルが見つからない

- AIに「どのファイルを作りましたか」と聞く
- 作業フォルダが `vibe_coding_test` になっているか確認する

### ブラウザでアプリを開けない

次のように聞いてください。

```prompt
作成したWebアプリをブラウザで開く方法を、初心者向けに1ステップずつ教えてください。
```

## 12. 次のステップ

最初の練習ができたら、次のような使い方に進みます。

1. 1日1回、短い作業をAIに依頼する
2. 成功したプロンプトをメモして再利用する
3. 作られたファイルを必ず確認する
4. 慣れてきたらGitやGitHubを学ぶ
5. 必要になったらCLI版のツールを試す

プロンプトを改善したい場合は、別のチャット画面で次のように聞くと便利です。

~~~Markdown
以下のプロンプトを、初心者にもわかりやすく、AIコーディングエージェントに伝わりやすい形に改善してください。
改善後のプロンプトは、コピーしやすいようにコードブロックで提示してください。

```
# ここにプロンプトを入力する
```
~~~

## おまけ: CLI版を使いたい人へ

ターミナルに慣れている人は、CLI版のツールも使えます。

- Codex CLI
- Claude Code CLI
- Gemini CLI

CLI版は便利ですが、初心者の最初の1回には必須ではありません。  
まずはデスクトップアプリやWeb版で、練習用フォルダだけを使って安全に試すことをおすすめします。
