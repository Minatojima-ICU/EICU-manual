# バイブコーディング練習ガイド（初心者向け）

このガイドは、バイブコーディング（コードを書かないで開発すること）を初めてやってみることを目的としています。  
対象者は、`npm` やターミナルを初めて使う方向けです。  
「最初の1回を安全に完了する」ことをゴールにしています。

## 1. この練習でできるようになること

- ターミナルを開いてコマンドを実行できる
- `Node.js` と `npm` をインストールできる
- CLIでLLM（Codex, Gemini CLI, Claude Code）を起動できる
- 短い指示文（プロンプト）で簡単な作業を依頼できる

## 2. 事前準備

### 必要なもの

- インターネット接続
- PC（Windows または Mac、Linux）
- メールアドレス（各サービス登録用）
- 30〜60分の作業時間

### 注意
- ChatGPT plus/Google AI pro/Claude proのユーザーは、新たに課金する必要はありません。ただし、上限があるので、普段使いをしている人は気をつけてください (特にClaude pro)。
- 2026/03/12現在、CodexとGeminiCLIは無料枠でも利用可能です。Claude Codeは無料枠はありません。
- **重要** API利用は課金が発生します（従量課金）。最初はChatGPT plus/Google AI pro/Claude pro、もしくはそれぞれの無料枠を利用すると良いでしょう。
- 病院から支給されているPCの場合は、院内ルールに従ってください。

## 3. ターミナルとは

ターミナルは、文字でPCに命令するアプリです。  
最初は「コピーして貼り付けて実行」で問題ありません。

### 開き方

- Mac: `Spotlight`で「ターミナル」を検索して開く
- Windows: `スタート`で「PowerShell」または「Windows Terminal」を検索して開く

### 最初の確認コマンド

```bash
echo "hello"
```

`hello`と表示されたらOKです。

### ターミナルの練習
[ターミナルの練習ページ](./terminal_practice.md) を先に完了してから進んでください。


## 4. Node.js と npm のインストール

CLIツールは`npm`経由で入れることが多いため、先に準備します。  
[詳細な手順はこちら](https://nodejs.org/ja/download) * 一行ずつコピペして実行してください。

1. Node.js公式サイトで`LTS`版をインストール  
2. ターミナルを再起動  
3. 次を実行して確認

```bash
node -v
npm -v
```

どちらもバージョン番号が出ればOKです。

## 5. CLI用LLMの設定

使いたいツールを1つ選んで進めてください。  
（慣れるまでは1つに絞るのがおすすめ）


### 6-2. Gemini CLIの場合

## 8. うまくいかないとき（よくある問題）

### `command not found`

- 原因: インストール未完了、またはターミナル再起動前
- 対処: インストール確認後、ターミナルを閉じて開き直す

### `npm: not found`

- 原因: Node.jsが入っていない
- 対処: Node.js（LTS）を再インストールして`node -v`/`npm -v`を確認

### 権限エラー（permission denied）

- 原因: 実行権限や保存先の制限
- 対処: 管理者権限のターミナルで再実行、または保存先フォルダを変更


## 9. 初めてのバイブコーディング
次のプロンプトをLLMに渡して、簡単なアプリを作ってみましょう。

```prompt.txt
## 課題
ARDS患者の目標一回換気量を計算して表示するwebアプリを作りたい。

## 内容
以下の条件にあうようにwebアプリを作成してください。

## 条件
- javascriptで記述してください
- `身長 (cm)`と`性別 (m or f)`を入力したら、6-8 ml/kg (IBW)の値に合致するように、適切な一回換気量の範囲を表示してください。
- 以下のDevineの式を使ってください。
  - Male: IBW (kg) = 50 + 2.3 × ((height_cm / 2.54) − 60)
  - Female: IBW (kg) = 45.5 + 2.3 × ((height_cm / 2.54) − 60)
```

できたアプリを実行する方法がわからなければ、LLMに聞いてください。変更が必要な場合は、追加でLLMに指示してください。

アプリ作成後、各ツールを終了するには
```bash
/quit
```
を入力してください。


## 10. 安全ルール（重要）

- 意味がわからないコマンドは実行しない
- `rm -rf`など削除系コマンドは特に慎重に扱う
- APIキーやパスワードをチャットに貼らない
- エラーが出たら、エラーメッセージをそのままLLMに見せて相談する


## 11. 次のステップ
以下のようなことを行い、慣れていってください。

1. 1日1回、短い作業をLLMに依頼する
2. 一旦命令のためのプロンプトを作った後に、次のようにLLMに聞いて、より良いプロンプトに変更する（この場合、別で開いたチャット形式のLLMツール (ChatGPTなど) に聞くとやりやすいです）。

~~~Markdown
以下のプロンプトを、適切な構造化を実施し、洗練させて、"~~~"で括って、私にコードブロックで提示してください。

```
# ここにプロンプトを入力する
```
~~~

3. 成功したプロンプトをメモして再利用する
4. `AGENTS.md` を利用する
5. ツールを立ち上げた後、"/" をタップしてツールの設定を使いやすいように変更する
